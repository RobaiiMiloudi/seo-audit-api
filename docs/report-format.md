# Report Format

When polling `GET /api/audit/:jobId` returns `"status": "completed"`, the `result` object will contain the full SEO report.

The report is assembled by the audit worker from three focused module outputs:

- `scraper` provides the `data` object.
- `performance` provides the `lighthouse` object.
- `audit.scorer` provides `score`, `grade`, and `suggestions`.

## High-Level Metrics

* **`score`**: A number out of 100 representing the overall SEO health. Deductions are made based on the severity of found issues.
* **`grade`**: A letter grade (A, B, C, D, or F) derived from the score.
* **`suggestions`**: An array of actionable advice generated by the API. Read these to understand exactly *why* points were deducted and how to fix the website.
* **`lighthouse`**: Performance metrics from PageSpeed/Lighthouse. This can be `null` if the external performance analysis fails while scraping still succeeds.
